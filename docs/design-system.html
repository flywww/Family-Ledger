Principles

Design decisions

Use these rules to keep the app private, financial, family-scale, monthly, and correctable.

Trust over decoration

Flat surfaces, subtle borders, stable controls, and no ornamental gradients inside authenticated workflows.

Dashboard first

Optimize for scanning, comparing, filtering, and correcting data before explanatory or promotional content.

Data confidence

Values should reveal freshness, estimation, failure, manual edits, and pending refresh states.

Visualization

Chart rules

Use line charts for time, bars for comparison, and donut charts only for simple composition.

Net worth over time

Line chart with readable currency tooltips and a table alternative for precise review.

Category comparison

Bar chart for precision. Avoid pie charts when similar values need close comparison.

Refresh health

Status list or segmented count cards. Pair color with labels and recovery actions.

Patterns

App structure

The authenticated product should feel like an operating surface, not a marketing site.

Use this quick pass before shipping UI changes in app and component surfaces.

Uses semantic tokens for surfaces, text, borders, and focus rings.
Financial values are right-aligned and use tabular numerals.
Status uses color plus text, icon, tooltip, or accessible label.
Hover, focus, loading, empty, and error states are defined.
